# 필기 정리

## 정보통신과 네트워크

키워드
  - 전송매체(Medium): 정보를 보낼 때 사용하는 물리적 경로 및 수단
  - 프로토콜: 통신을 원활하게 하기 위한 `상호 간의 규칙 혹은 약속`
  - 네트워크의 장점: 데이터의 공유 가능, `분산` 저장 -> 부하 방지, 위험 최소화
  - 토폴로지(Topology): 목적과 형태에 따른 다양한 `네트워크 설계 방법` 
    - 망형(Mesh): `모든` 디바이스 점대점 연결 / 복잡도 높음 / `독립적`
    - 성형(Star): 중앙 시스템 `의존` / 효율적 / 견고함이 떨어짐
    - 트리형(Tree): 방사형 / 상위 노드에 `의존`적
    - 버스형: 간단한 구조 / 터미네이터
    - 링형 : 양 끝단이 서로 연결 / `특정 공간 내`에서 네트워크 구축 시 이용

## 네트워크의 분류

- 데이터 `전송방식`에 따른 분류
  1. 회선 교환방식(아날로그)
    - 전화 교환수를 생각하자
    - 독립적 회선 -> `보안`이 강력
  2. 패킷 교환방식(디지털)
    - 데이터만 이동
    - `switch`를 이용, 효율적인 경로로 이동 가능
    - 전송 효율이 높고 데이터 처리량이 많다.
- 데이터 `통신방식`에 따른 분류
  - `유니`캐스트: 1대1 통신
  - `브로드`캐스트: 로컬`LAN` 전체와 1:N 통신
    - LAN(`Local` Area Network): 10KM 이내의 토폴로지 통신망
    - MAN(`Metropolitan` Area Network): 도시 규모의 통신망
    - WAN(`Wide` Area Network): 국가나 대륙 
  - `멀티`캐스트: 특정 그룹과 1:N 통신
- 전송매체에 따른 분류
  - 이중나선 케이블
    - UTP: 외부신호의 간섭을 막기 위한 별도의 처리는 없지만 `저렴`해서 가장 `대중적`
    - FTP: 간섭을 막기 위해 피복을 씌워서 보호
    - STP: 각각의 꼬임선 까지도 피복을 씌워서 간섭을 막음
    - CATEGOTY 5: 100Mbps
  - 동축 케이블
    - 이중나선 케이블과 달리 `큰 대역폭`을 지원하여 `한 번에 더 많은 데이터`를 전송할 수 있음
    - 속도는 이중나선 케이블에 비해 느림. 10Mbps
  - 광섬유 케이블
    - 유리섬유의 한쪽 끝에서 주파수 신호를 빛의 펄스로 변환
    - 전송하는 빛의 `각도`를 자유자재로 변환하여 한 번에 많은 데이터를 전송할 수 있으며 외부 신호의 간섭을 적게 받는다. 
    - 매우 많은 정보량 & 장거리 전송 & 손실이 매우 적다.
    - 단점: `단방향` / 구축 비용이 비쌈 / 끊어지기 쉽기 때문에 매립 설치 -> 비용 높음
  - 무선 전송 매체
    - 공기 중으로 전파되는 전자파를 매개로 데이터를 전송

## 프로토콜
정의: 정보통신을 원활하게 하기 위한 상호 간의 규칙 혹은 약속
  - 주요 프로토콜: TCP/IP(인터넷 프로토콜)

프로토콜의 3요소
  - 구문(문법): 아날로그 또는 디지털 등의 표현을 정의
    - ASCII CODE & UNICODE: 모든 문자마다 고유한 코드 값을 정의
    - UNICODE를 인코딩(해석)하기 위한 방식으로 주요한 것으로 UTF-8/16 등이 있다.
  - 의미: 데이터 무결성 검사 등을 위한 제어 정보를 규정
  - 타이밍: 송수신자간 신호 지속 시간 / 신호 순서등을 조정하여 데이터 손실을 방지

프로토콜의 기능
1. 주소 지정(Addressing)
  - 네트워크의 노드 마다 서로 식별할 수 있는 고유 번호를 부여
  -`주소 == IP / 이름 == MAC`
2. 단편화(Fragmentation)와 재조립(Reassembly)
  - 큰 데이터의 경우 보내기 전 쪼개서 보내고 받을 때 재조립
3. 순서 지정(Sequencing)
  - index를 지정하는 것처럼 `데이터 패킷에 순서를 부여`하여 순서대로 정보가 교환되도록 함
  - 연결지향형에서만 사용함
    - 연결지향형? 데이터를 전송하는 측과 데이터를 전송받는 측에서 `전용의 데이터 전송 경로`를 만든다.
4. 데이터 흐름 제어(Data Flow Control)
  - `수신자` 측에서 데이터 전송량이나 속도를 조절함
  - `슬라이딩 윈도우` 등이 대표적
5. 연결 제어(Connection Control)
  - 연결 지향형 데이터 전송 -> 연결 설정 / 데이터 전송 / 연결 해제
6. 캡슐화(Encapsulation)
  - 네트워크의 각 계층별로 사용할 정보를 관리하기 편하도록 구분
7. 오류 제어(Error Control)
  - 오류 발생을 검출, 재전송을 요구하거나 자체 복구
8. 동기화(Synchronization)
  - 데이터 전송 시 타이밍이나 윈도우 크기 등을 `합의`하는 것.
9. 멀티플렉싱
  - `한 번에 여러 정보`를 통신할 수 있도록 `가상의 회선` 설정
10. 전송 서비스
  - `차등`을 두고 전송을 결정

프로토콜의 전송 방식
1. 비트 방식
2. 바이트 방식
3. 문자 방식

프로토콜의 특성
1. 직접과 간접
  - 직접적: 두 통신 시스템이 점대점 연결
  - 간접적: 여러 개의 통신 시스템과 하나의 커다란 시스템간의 연결

## 근거리 통신(LAN) 프로토콜
1. 알로하 프로토콜
  - 세계 최초로 경쟁법을 도입한 다원 접속 프로토콜.
  - 향후 `이더넷(유선 LAN)`의 시초
  - `송신 측(중앙 제어국)`을 중심으로 하는 `성형 토폴로지`
2. 슬롯 알로하 프로토콜
  - 전송 데이터를 `슬롯화` / 각 슬롯의 `시작점에서만 패킷을 전송`할 수 있도록 개선한 프로토콜
  - 충돌 확률이 줄어들어 순수 알로하 프로토콜에 비해 전송 효율이 `2배` 높다.
3. CSMA/CD(Carrier Sense Multiple Access With Collision Detection)
  - 슬롯 알로하의 발전된 형태
  - `테스트` 패킷 전송 / 다른 전송이 선행되어 있다면 대기 / 비어있을 때 재전송 / 여러 대의 단말이 LAN 선에 `충돌 없이` 접촉할 수 있게 하는 프로토콜
4. IEEE LAN 표준
  - `802.x`는 LAN의 국제 표준안 규격
  - 802.2: 논리 링크 제어(LLC: Logical Link Control)
    - 여러 다양한 매체접속제어 방식 간의 `차이를 보완`해주는 역할
    - 여러 다른 MAC 부계층 프로토콜을 사용하게 하여 망의 구조,구성에 `상관없이` 통신을 가능하게 해준다.
  - 802.3: CSMA/CD, Ethernet
  - 802.11 a/b/g/n/ac: 무선 네트워크
  - 802.15.1: 블루투스
  - 802.15.4: 직비(ZigBee)
    - 소형, 저전력 `디지털 라디오`를 이용해 `개인통신망`을 구성하여 통신하기 위한 규격. `스마트홈`에 주로 사용.
    - 기본적으로 `초저전력 환경`에서 `대량의 기기`와 통신하기 위해 개발
  - 802.16: 광대역 무선 접속(WiMAX)
  - 802.16e: 광대역 무선 접속(모바일)

## 정보통신 이론

채널 용량
  - 어떤 전송 매체의 최대 정보 전송 능력

신호(Signal)
  - 아날로그: `연속적`으로 변하는 전자기적 신호
  - 디지털: 0과 1로 표현(이산적)되는 데이터 신호

정보의 전송 방식
1. 신호의 종류에 따른 분류
  - 아날로그: 연속적 -> 잡음 발생 가능성이 높음
  - 디지털: 이산적 -> 비교적 잡음 발생 가능성이 낮고 에러가 적다.
2. 데이터의 전송 방향에 따른 분류
  - 단방향(Simplex) 통신: TV, RADIO
  - 반이중(Half-Duplex) 통신: 양방향 통신이 가능하지만 동시에는 불가능. 무전기 등
  - 전이중(Full-Duplex) 통신: 전화
3. 데이터의 전송 단위에 따른 분류
  - 직렬: 여러 개의 데이터 / 하나의 회선 / 순서대로 -> 동기식 전송방식
    - 장거리 전송에 적합하지만 전송 속도가 느리다.
  - 병렬: 여러 개의 데이터 / 여러 회선 / 동시에 -> 비동기식 전송방식
    - 각 회선의 순서로 인한 에러 발생 가능성이 높지만 직렬에 비해 빠르다.
    - 다만 직렬에 비해 전송 비용이 높다.
4. 데이터의 동기화 여부에 따른 분류
  - 비동기식: 스타트 비트와 스탑 비트를 사용 / 한 문자 단위로 데이터를 전송
    - 각 문자 사이에 `idle`이 존재해 전송 효율이 낮아져 속도가 느리다.
  - 동기식: 한 문자를 쪼개어 / 빠르게 전송 / 효율이 높아 빠르다.
  - 통신 속도/처리량 관련 주요 개념
    - BPS(Bit Per Second): 1초당 전송할 수 있는 비트의 수 

변조: 아날로그와 디지털 신호의 변환
1. 아날로그 -> 아날로그
  - AM: `신호의 높낮이`로 정보의 차이 표현
  - FM: `진폭의 밀도`로 정보의 차이를 표현
  - PM: `신호의 모양`으로 정보의 차이를 표현
2. 아날로그 -> 디지털
  - PCM(Purse Code Modulation): (송신)아날로그 -> `펄스` -> (수신) 디지털 -> 송신 당시의 아날로그와 유사하게 변조
    - 표본화(Sampling): 아날로그 파형을 일정한 시간 폭으로 `나누어` 비연속적인 `직사각형으로 분할`하는 단계
      - 해당 시간 폭 동안의 여러 값을 `하나의 대표적인 값`으로 바꿔주는 단계
    - 양자화(Quantumization): 표본화된 값들을 `계산하기 쉬운 일정한 값으로 균일화`하는 단계. 예를들어 소수점을 반올림하여 정수값으로 변환하는 것.
    - 부호화(Encoding): 양자화된 값을 `2진수`로 변환.
    - 복호화(Decoding): `2진수` 값 -> `펄스` 신호 변환.
    - 필터링(Filtering): 펄스간을 `부드럽게 이어` 본래의 아날로그 신호와 `유사하게 변환` 
3. 디지털 -> 아날로그
  - ASK(진폭 변이 변조): 위아래 진폭의 차이로 0과1 구분
  - FSK(주파수 변이 변조): 좌우 진폭의 밀도 차이로 0과1 구분
  - PSK(위상 변이 변조): `360도를 한 주기로` 하는 하나의 그래프를 쪼개어 나타나는 `여러 가지 모양(위상)`에 2진법의 숫자를 대입하여 변조.
  -  위상을 너무 많이 쪼개면 각 위상 간의 정확한 판별이 어려워 신호가 지연될 수 있다. 
4. 디지털 -> 디지털
  - 단극형(Unipolar): 극성이 발생하면 1, 아니면 0
  - 극형(Polar): 양극과 음극 두 개의 레벨을 사용
  - 양극형(Bipolar): 양극 / 0 / 음극 3개의 전압을 사용
    - 0은 0, 양극과 음극은 1을 나타낸다.

통신 관련 기술
1. 흐름제어(Flow Control)
  - 통신을 주고받는 두 개의 컴퓨터 간에 `데이터 처리 속도`가 달라서 발생하는 문제들을 해결하기 위한 방법
  대표적인 방법
  슬라이딩 윈도우
    - 수신측이 설정한 `윈도우 크기에 대한 부분` 만큼은 확인 메시지(ACK)가 없어도 데이터 전송을 `확신`할 수 있다.
    - 만약 수신측이 수신 예정이던 데이터를 받지 못할 경우 송신측으로 NAK 메시지를 보내고, 송신측에서는 보내는 `메시지의 크기`를 줄인다.
    - 동적으로 윈도우의 크기를 조절하여 송수신측 데이터 처리 속도 차이를 조절
2. 오류제어
  
  전송오류의 종류
    1. 감쇠(Attenuation): 전송 도중 `신호가 약해져` 정확히 판단할 수 없는 경우
    2. 지연 왜곡(Delay Distortion): 전송이 늦어지는 경우
    3. 잡음(Noise): 주변 전자기기의 영향으로 인한 목표 신호의 왜곡
    4. 혼선(Crosstalk): 전기 신호적 결합으로 다른 회선에 영향을 미치는 경우
  
  오류 검출 방법
    1. 패리티 비트 검사: 전송 데이터마다 `패리티 비트`를 추가하여 비트 개수의 `홀/짝 여부`로 오류를 확인.
      - 총 8개의 비트 중 뒤의 7개 비트를 확인, `검사 방법에 따라` 홀/짝으로 맞추는 방식으로 오류 여부 확인
      - 한계: 8개의 비트에서 `2번`의 오류가 발생하면 판별 불가
    2. 블록 합 검사(Block Sum Check): 패리티 비트 검사의 한계 극복을 위해 수평과 수직 2차원적으로 블록을 합쳐 검사. 하지만 한계도 존재함!
    3. 순환 중복 검사(Cyclic Redundancy Check): 정확한 에러 검출을 위해 다항식을 사용
  
  오류 제어의 종류
     - 기본형(Stop and Wait ARQ): `수신자`가 신호를 보낼때까지 대기
     - 연속적(Go back N ARQ): 에러가 발생한 블록 이후의 모든 블록을 재전송
     - 선택적(Selective ARQ): 수신측이 `에러 발생~` 하고 재전송 요청이 있으면 에러가 발생한 블록 중 해당 프레임만 다시 전송.
     - 적응적(Adaptive ARQ): 에러가 발생하면 전송하는 데이터의 프레임 크기를 조정

## 고속LAN 및 광대역 기술

고속이더넷: 10mbps -> 100mbps / 100base-T / 포맷과 프로토콜(CSMA/CD)이 동일
  - 통신망은 보통 성형으로 구축됨.
  - 기가비트 이더넷: 100Mbps -> 1Gbps / 1000base-X / 포맷과 프로토콜(CSMA/CD) 동일

토큰링
- 토큰패싱 접근법을 이용한 네트워크 기술
- 각 노드에 정보 교환을 위한 `권한(토큰)을 부여`, 해당 토큰을 보유하고 있는 동안만 해당 노드가 `독점적`으로 데이터를 전송할 수 있는 형태
- 링 형태의 토폴로지 네트워크 / `충돌이 없다`

FDDI: 미국 ANSI(미국국가`표준`협회)에서 표준화한 프로토콜. 토큰패싱 접근법을 이용.

## OSI 계층

### 개요
1. OSI란? Open System Interconnection, 개방형 시스템 간의 상호접속
2. ISO에서 제정. 개방형 시스템에서 `서로 다른 네트워크`에 속한 디바이스들이 다양한 상황에 `관계없이 항상 같은 형태`로 데이터를 교환할 수 있도록 정한 국제표준. 
3. 7Layers   계층       /   데이터 단위  / 사용 디바이스  
  - 1 Layer: 물리 계층 / Reapeater, Hub / Bit
  - 2 Layer: 데이터 링크 계층 / Bridge, Switch / Frame
  - 3 Layer: 네트워크 계층 / Router, Layer 3 Switch / Packet or Datagram
  - 4 Layer: 전송 계층 / / Record or Segment or Segment or Gateway
  - --------------------------------------------------------------------- ↑ 운영체제 내부에서 데이터 전송을 다룸
  - 5 Layer: 세션 계층 / / SoftWare Gateway
  - 6 Layer: 표현 계층 / / SoftWare Gateway
  - 7 Layer: 응용 계층 / / SoftWare Gateway
  - 7 Layer이 가장 일반 사용자와 가까운 계층.
4. 계층화의 장점
  - 각 계층의 독립성 보장
  - 독립성으로 인해 오류 대응 용이
5. 계층별 특징
  1. 1 Layer(물리 계층)
    - 물리적 장비
    - 비트(0or1) 기반 데이터 전송
    - 주요장비: 케이블 / 허브 / 리피터(전송거리가 먼 경우 증폭용도로 활용)
  2. 2 Layer(데이터 링크 계층)
    - 송수신을 `확인`하여 정보 교환이 가능한지`(오류 검출 / 흐름 제어)` 검사
    - `네트워크 계층(3 Layer)`에서 붙은 IP헤더를 활용, `Mac Address`를 구해 송수신자의 주소를 파악.
  3. 3 Layer(네트워크 계층)
    - 송수신자 간 `경로`를 설정, 목적지까지 정보를 전달.
    - 논리적 주소를 물리적 주소로 `변환`
    - 2 Layer(데이터 링크 계층)와 달리 네트워크 주소`(IP주소)`까지 참조하여 `최적경로`를 설정 
    - 주요 프로토콜
      - IP(Internet Protocol)
      - ICMP(Internet Control Message Protocol)
      - IGMP(Internet Group Message Protocol)
      - ARP(Address Resolution Protocol): 주소를 알아내는 프로토콜
      - RARP(Reverse Address Resolution Protocol)
  4. 4 Layer(전송 계층)
    - 송수신자 간 `논리적 연결`을 담당하는 계층
    - 오류가 없는지 검출
    - 주요 프로토콜
      - TCP(Transmission Control Protocol)
      - UDP(User Datagram Protocol)
        - Datagram? 패킷의 일종으로 송수신자간 경로 지정을 위한 정보를 포함함.
      - SCTP(Stream Control Transmission Protocol) 
  5. 5 Layer(세션 계층)
    - 송수신자간 통신을 위한 `동기화 신호를 교환`하여 연결 세션을 구축 / 유지 / 설정 / 종료함
      - 동기화 신호? 제대로된 정보 처리를 위해 통신 양측이 공통된 시작점을 합의하는 것.
      - 재동기? 동기점을 이용하여 전송 오류를 복구하는 것.
    - 해당 계층에서 세션의 연결 방식(단방향 / 반이중 / 전이중)을 결정한다. 
  6. 6 Layer(표현 계층)
    - 앱에서 보내온 메시지를 전송 가능한 형태로 변환(코드화)
    - 데이터를 변환한다고 하여 문맥 계층이라고도 부름.
    - `암호화`를 직접적으로 수행하는 계층
  7. 7 Layer(응용 계층)
    - 앱 차원에서 송수신 데이터가 사용할 수 있는 통로를 제공(`추상화`)
    - 정보의 단위: Message
    - 주로 사용되는 프로토콜
       - HTTP: 인터넷 통신을 위한
       - FTP: 파일 전송을 위한
       - SMTP: 메일 발송을 위한
