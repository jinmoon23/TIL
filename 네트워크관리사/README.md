# 필기 정리

## 정보통신과 네트워크
키워드
    - 전송매체(Medium): 정보를 보낼 때 사용하는 물리적 경로 및 수단
    - 프로토콜: 통신을 원활하게 하기 위한 상호 간의 규칙 혹은 약속
    - 네트워크의 장점: 데이터의 공유 가능, 분산 저장 -> 부하 방지, 위험 최소화
    - 토폴로지(Topology): 목적과 형태에 따른 다양한 네트워크 설계 방법 
      - 망형(Mesh): 모든 디바이스 점대점 연결 / 복잡도 높음 / 독립적
      - 성형(Star): 중앙 시스템 의존 / 효율적 / 견고함이 떨어짐
      - 트리형(Tree): 방사형 / 상위 노드에 의존적
      - 버스형: 간단한 구조 / 터미네이터
      - 링형 : 양 끝단이 서로 연결 / 특정 공간 내에서 네트워크 구축 시 이용

## 네트워크의 분류
- 데이터 전송방식에 따른 분류
  1. 회선 교환방식(아날로그)
    - 전화 교환수를 생각하자
    - 독립적 회선 -> `보안`이 강력
  2. 패킷 교환방식(디지털)
    - 데이터만 이동
    - switch를 이용, 효율적인 경로로 이동 가능
    - 전송 효율과 데이터 처리의 양이 많다.
- 통신 방식에 따른 분류
  - 유니캐스트: 1대1 통신
  - 브로드캐스트: 로컬`LAN` 전체와 1:N 통신
    - LAN(Local Area Network): 10KM 이내의 토폴로지 통신망
    - MAN(Metropolitan Area Network): 도시 규모의 통신망
    - WAN(Wide Area Network): 국가나 대륙 
  - 멀티캐스트: 특정 그룹과 1:N 통신
- 전송매체에 따른 분류
  - 이중나선 케이블
    - UTP: 외부신호의 간섭을 막기 위한 별도의 처리는 없지만 저렴해서 가장 대중적
    - FTP: 간섭을 막기 위해 피복을 씌워서 보호
    - STP: 각각의 꼬임선 까지도 피복을 씌워서 간섭을 막음
    - CATEGOTY 5: 100Mbps
  - 동축 케이블
    - 이중나선 케이블과 달리 큰 대역폭을 지원하여 한 번에 더 많은 데이터를 전송할 수 있음
    - 속도는 이중나선 케이블에 비해 느림. 10Mbps
  - 광섬유 케이블
    - 유리섬유의 한쪽 끝에서 주파수 신호를 빛의 펄스로 변환
    - 전송하는 빛의 각도를 자유자재로 변환하여 한 번에 많은 데이터를 전송할 수 있으며 외부 신호의 간섭을 적게 받는다. 
    - 매우 많은 정보량 & 장거리 전송 & 손실이 매우 적다.
    - 단점: 단방향 / 구축 비용이 비쌈 / 끊어지기 쉽기 때문에 매립 설치 -> 비용 높음
  - 무선 전송 매체
    - 공기 중으로 전파되는 전자파를 매개로 데이터를 전송

## 프로토콜
정의: 정보통신을 원활하게 하기 위한 상호 간의 규칙 혹은 약속
  - 주요 프로토콜: TCP/IP(인터넷 프로토콜)

프로토콜의 3요소
  - 구문(문법): 아날로그 또는 디지털 등의 표현을 정의
    - ASCII CODE & UNICODE: 모든 문자마다 고유한 코드 값을 정의
    - UNICODE를 인코딩(해석)하기 위한 방식으로 주요한 것으로 UTF-8/16 등이 있다.
  - 의미: 데이터 무결성 검사 등을 위한 제어 정보를 규정
  - 타이밍: 송수신자간 신호 지속 시간 / 신호 순서등을 조정하여 데이터 손실을 방지

프로토콜의 기능
1. 주소 지정(Addressing)
  - 네트워크의 노드 마다 서로 식별할 수 있는 고유 번호를 부여
  -`주소 == IP / 이름 == MAC`
2. 단편화(Fragmentation)와 재조립(Reassembly)
  - 큰 데이터의 경우 보내기 전 쪼개서 보내고 받을 때 재조립
3. 순서 지정(Sequencing)
  - index를 지정하는 것처럼 `데이터 패킷에 순서를 부여`하여 순서대로 정보가 교환되도록 함
  - 연결지향형에서만 사용함
    - 연결지향형? 데이터를 전송하는 측과 데이터를 전송받는 측에서 `전용의 데이터 전송 경로`를 만든다.
4. 데이터 흐름 제어(Data Flow Control)
  - `수신자` 측에서 데이터 전송량이나 속도를 조절함
  - `슬라이딩 윈도우` 등이 대표적
5. 연결 제어(Connection Control)
  - 연결 지향형 데이터 전송 -> 연결 설정 / 데이터 전송 / 연결 해제
6. 캡슐화(Encapsulation)
  - 네트워크의 각 계층별로 사용할 정보를 관리하기 편하도록 구분
7. 오류 제어(Error Control)
  - 오류 발생을 검출, 재전송을 요구하거나 자체 복구
8. 동기화(Synchronization)
  - 데이터 전송 시 타이밍이나 윈도우 크기 등을 `합의`하는 것.
9. 멀티플렉싱
  - `한 번에 여러 정보`를 통신할 수 있도록 `가상의 회선` 설정
10. 전송 서비스
  - `차등`을 두고 전송을 결정

프로토콜의 전송 방식
1. 비트 방식
2. 바이트 방식
3. 문자 방식

프로토콜의 특성
1. 직접과 간접
  - 직접적: 두 통신 시스템이 점대점 연결
  - 간접적: 여러 개의 통신 시스템과 하나의 커다란 시스템간의 연결

## 근거리 통신(LAN) 프로토콜
1. 알로하 프로토콜
  - 세계 최초로 경쟁법을 도입한 다원 접속 프로토콜.
  - 향후 `이더넷(유선 LAN)`의 시초
  - `송신 측(중앙 제어국)`을 중심으로 하는 `성형 토폴로지`
2. 슬롯 알로하 프로토콜
  - 전송 데이터를 `슬롯화` / 각 슬롯의 `시작점에서만 패킷을 전송`할 수 있도록 개선한 프로토콜
  - 충돌 확률이 줄어들어 순수 알로하 프로토콜에 비해 전송 효율이 `2배` 높다.
3. CSMA/CD(Carrier Sense Multiple Access With Collision Detection)
  - 슬롯 알로하의 발전된 형태
  - `테스트` 패킷 전송 / 다른 전송이 선행되어 있다면 대기 / 비어있을 때 재전송 / 여러 대의 단말이 LAN 선에 `충돌 없이` 접촉할 수 있게 하는 프로토콜
4. IEEE LAN 표준
  - `802.x`는 LAN의 국제 표준안 규격
  - 802.2: 논리 링크 제어(LLC: Logical Link Control)
    - 여러 다양한 매체접속제어 방식 간의 `차이를 보완`해주는 역할
    - 여러 다른 MAC 부계층 프로토콜을 사용하게 하여 망의 구조,구성에 `상관없이` 통신을 가능하게 해준다.
  - 802.3: CSMA/CD, Ethernet
  - 802.11 a/b/g/n/ac: 무선 네트워크
  - 802.15.1: 블루투스
  - 802.15.4: 직비(ZigBee)
    - 소형, 저전력 `디지털 라디오`를 이용해 `개인통신망`을 구성하여 통신하기 위한 규격. `스마트홈`에 주로 사용.
    - 기본적으로 `초저전력 환경`에서 `대량의 기기`와 통신하기 위해 개발
  - 802.16: 광대역 무선 접속(WiMAX)
  - 802.16e: 광대역 무선 접속(모바일)

## 정보통신 이론

채널 용량
  - 어떤 전송 매체의 최대 정보 전송 능력

신호(Signal)
  - 아날로그: `연속적`으로 변하는 전자기적 신호
  - 디지털: 0과 1로 표현(이산적)되는 데이터 신호

정보의 전송 방식
1. 신호의 종류에 따른 분류
  - 아날로그: 연속적 -> 잡음 발생 가능성이 높음
  - 디지털: 이산적 -> 비교적 잡음 발생 가능성이 낮고 에러가 적다.
2. 데이터의 전송 방향에 따른 분류
  - 단방향(Simplex) 통신: TV, RADIO
  - 반이중(Half-Duplex) 통신: 양방향 통신이 가능하지만 동시에는 불가능. 무전기 등
  - 전이중(Full-Duplex) 통신: 전화
3. 데이터의 전송 단위에 따른 분류
  - 직렬: 여러 개의 데이터 / 하나의 회선 / 순서대로 -> 동기식 전송방식
    - 장거리 전송에 적합하지만 전송 속도가 느리다.
  - 병렬: 여러 개의 데이터 / 여러 회선 / 동시에 -> 비동기식 전송방식
    - 각 회선의 순서로 인한 에러 발생 가능성이 높지만 직렬에 비해 빠르다.
    - 다만 직렬에 비해 전송 비용이 높다.
4. 데이터의 동기화 여부에 따른 분류
  - 비동기식: 스타트 비트와 스탑 비트를 사용 / 한 문자 단위로 데이터를 전송
    - 각 문자 사이에 `idle`이 존재해 전송 효율이 낮아져 속도가 느리다.
  - 동기식: 한 문자를 쪼개어 / 빠르게 전송 / 효율이 높아 빠르다.
  - 통신 속도/처리량 관련 주요 개념
    - BPS(Bit Per Second): 1초당 전송할 수 있는 비트의 수 

변조: 아날로그와 디지털 신호의 변환
1. 아날로그 -> 아날로그
  - AM: `신호의 높낮이`로 정보의 차이 표현
  - FM: `진폭의 밀도`로 정보의 차이를 표현
  - PM: `신호의 모양`으로 정보의 차이를 표현
2. 아날로그 -> 디지털
  - PCM(Purse Code Modulation): (송신)아날로그 -> `펄스` -> (수신) 디지털 -> 송신 당시의 아날로그와 유사하게 변조
    - 표본화(Sampling): 아날로그 파형을 일정한 시간 폭으로 `나누어` 비연속적인 `직사각형으로 분할`하는 단계
      - 해당 시간 폭 동안의 여러 값을 `하나의 대표적인 값`으로 바꿔주는 단계
    - 양자화(Quantumization): 표본화된 값들을 `계산하기 쉬운 일정한 값으로 균일화`하는 단계. 예를들어 소수점을 반올림하여 정수값으로 변환하는 것.
    - 부호화(Encoding): 양자화된 값을 `2진수`로 변환.
    - 복호화(Decoding): `2진수` 값 -> `펄스` 신호 변환.
    - 필터링(Filtering): 펄스간을 `부드럽게 이어` 본래의 아날로그 신호와 `유사하게 변환` 
3. 디지털 -> 아날로그
  - ASK(진폭 변이 변조): 위아래 진폭의 차이로 0과1 구분
  - FSK(주파수 변이 변조): 좌우 진폭의 밀도 차이로 0과1 구분
  - PSK(위상 변이 변조): `360도를 한 주기로` 하는 하나의 그래프를 쪼개어 나타나는 `여러 가지 모양(위상)`에 2진법의 숫자를 대입하여 변조.
  -  위상을 너무 많이 쪼개면 각 위상 간의 정확한 판별이 어려워 신호가 지연될 수 있다. 
4. 디지털 -> 디지털
  - 단극형(Unipolar): 극성이 발생하면 1, 아니면 0
  - 극형(Polar): 양극과 음극 두 개의 레벨을 사용
  - 양극형(Bipolar): 양극 / 0 / 음극 3개의 전압을 사용
    - 0은 0, 양극과 음극은 1을 나타낸다.
